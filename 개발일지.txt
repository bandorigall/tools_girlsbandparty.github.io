================================================================
걸즈밴드파티 (걸파 툴 모음) — 개발일지
================================================================
(레포: bandorigall/tools_girlsbandparty.github.io / GitHub Pages)
라이브: https://bandorigall.github.io/tools_girlsbandparty.github.io/

■ 배경/동기
  걸파 관련 도구가 여러 레포로 흩어져 있었다(순위분석, 월간랭킹, 마리나불캔).
  사용자가 한 곳에서 탭으로 오가게 하려고 이 허브 레포로 통합.
  기존 월간랭킹/마리나불캔 단독 레포는 이 사이트로 이관하고, 옛 주소는
  이 허브의 #monthly/#marina 로 리다이렉트하는 안내 페이지로 남겼다.

■ 통합 구조
  - index.html 이 상단 탭바 + iframe 지연 로딩으로 3개 하위 툴을 묶음.
    (탭 처음 누를 때만 iframe 생성 → 초기 로딩 가볍게. #딥링크 지원)
  - 탭 순서: 순위분석 → 월간랭킹 → 마리나불캔.
  - monthly/ · marina/ 는 이관된 정적 계산기(전체 페이지 CSS/JS가 서로 충돌해서
    병합 대신 iframe으로 격리한 것이 핵심 판단).
  - rank/ 만 서버측(Actions) 학습·예측이 붙는 핵심 도구.

■ 순위분석(rank) — 예측 방식 재구축 (핵심)
  - 문제: 기존 방식(직전 이벤트 1개에 GAM 외삽 + 잔차 ARIMA)이
    "막판 급주"(이벤트 막바지 컷 급상승)를 과소예측.
  - 해결: 과거 다수 이벤트 기반 '티어별 그래디언트 부스팅'으로
    log(최종컷/현재컷) 배율을 직접 회귀 → 막판 상승을 데이터에서 학습.
  - 학습 풀: 같은 티어 + 같은 이벤트 타입 우선, 표본<15면 티어 전체 폴백.
    (밴드 필터는 표본만 깎여 오히려 나빠져서 뺌.)
  - 불확실성: quantile 회귀로 안전컷(α=0.95)·하한(α=0.10) 동시 산출.
  - 검증(LOO 백테스트, 진행 85~95%): naive 막판오차 19.7% → GBM 2.7%,
    안전컷 실제 커버리지 ≈ 89%. (상위 memory "걸파 순위 예측툴" 참조)

■ 자동화 결정
  - 브라우저에서 모델을 돌리지 않는다 → GitHub Actions(rank.yml)가
    매일 00:00 KST에 현재 이벤트만 fetch+예측하여 data.json 커밋.
  - 과거 학습 데이터(curves.npz)를 레포에 동봉 → Actions가 과거를
    재수집하지 않고 현재 이벤트만 실시간 fetch(속도/안정성).
  - data.json 변경 있을 때만 커밋([skip ci]), concurrency로 중복 실행 방지.

■ 이력(시간순 삽질 포함)
  - 통합 방식 논의: 폴리레포 vs 모노레포. 이미 도구별 단독 레포가 굳어져 있어
    이 셋만 허브로 묶는 절충. "트래픽 몰리면?" 걱정은 정적+CDN 서빙이라 무의미.
  - github.io 네이밍 오해 정리: username.github.io 특별 서빙은 계정명과 같은
    레포 1개뿐. 나머지는 프로젝트 페이지(.../repo/)로 정상 동작. 기존 레포들이
    전부 xxx.github.io 로 명명돼 URL에 .github.io 가 두 번 들어가지만 기능 정상.
  - init 커밋: 탭 허브 + 3개 도구 통합.
  - 1차 push 거부: OAuth 토큰에 workflow 스코프가 없어 .github/workflows/rank.yml
    거부됨 → 워크플로만 커밋에서 빼고(git rm --cached + amend) 나머지 push,
    rank.yml 은 GitHub 웹 UI로 추가하여 우회.
  - 첫 Actions 실행 실패(exit 1): bestdori-api 가 HTTP 클라이언트(httpx/aiohttp)를
    자동 설치하지 않아 ImportError. 로컬은 이미 깔려 있어 재현 안 됐던 함정.
    → requirements.txt 에 httpx 명시 후 재실행 성공. data.json 정상 생성.
  - 구 레포 리다이렉트: marinafirecan / monthly_ranking_calculator 의 index.html 을
    JS location.replace + meta refresh(3s) + canonical + 수동 링크로 교체하여
    각각 이 허브의 #marina / #monthly 로 유도. 두 레포 push 완료.
  - git_auto.sh 추가: add→commit→pull→push 자동화. pull 을 push 앞에 둬서
    Actions 봇이 올린 data.json 갱신본과 충돌 방지(추가 직후 실제로 merge 발생).

■ 운영 메모
  - 순위 데이터는 Actions가 알아서 갱신(수동은 Actions 탭 workflow_dispatch).
  - 로컬에서 파일 고친 뒤엔 ./git_auto.sh 한 번이면 커밋/풀/푸시 끝.
  - 예측 코드 수정 시 rank/predictor.py, 검증은 experiment/ 백테스트(로컬 원본).
  - get_events.py 의 밴드/타입 매핑은 신규 밴드 추가 시 갱신 필요.
  - 워크플로 파일(.github/workflows/*.yml)을 로컬에서 고쳐 push 하려면 git 토큰에
    workflow 스코프 필요. 없으면 웹 UI로 수정.
  - tksqhddnfl5 계정은 bandorigall 레포에 admin 아님 → gh 로 재실행(dispatch)
    불가, 로그 열람은 가능.
